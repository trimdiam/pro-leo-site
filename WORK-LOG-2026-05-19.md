## 3. Photo-URL Save Bug Fix

**Problem:** Photo upload succeeded but the URL never persisted to Firestore.

**Root cause:** Firebase Storage download URLs contain `&` (for the auth token),
which broke the HTML when the teacher object was inlined into an `onclick`
attribute via `JSON.stringify`. The deserialized teacher object then had
`photoURL: undefined`.

**Fix:**
- The edit button now only passes the `docId`. `editTeacher()` looks up the
  full teacher object from `window._loadedTeacherDocs`.
- Replaced the hidden-input photo URL with a `window._teacherPhotoURL`
  module variable (reliable across async upload → save).
- Removed a recursive `_loadedTeacherDocs` inline update that would have
  stack-overflowed if triggered.

**File:** `app-logic.js`
