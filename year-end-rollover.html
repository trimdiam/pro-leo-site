🎓 Year-End Rollover

St. Francis De Sales School — advance the academic session safely. Every destructive step is gated and preview-first.

Sign in with your Admin account to continue.

Set the labels, then run a dry-run preview. The preview writes nothing — it only counts what each step would touch.

1Full backup

Export the entire Firestore project before anything else — this is the only thing that can fully restore the school's data if a later step goes wrong. Run in Cloud Shell:
gcloud firestore export gs://<your-bucket>/backups/2026-27-final

2Archive academic data

Copy session-bound academic collections into archive_2026-27_* so last year stays viewable in-app. Copy-only — the source data is not touched.

3Promote students

Advance each student's class; detained students stay; Class 10 moves to alumni. Roll numbers are left as-is (reassign manually after rollover).

4Clear & reset

Wipe the now-archived live academic collections and reset transient counters so the new year starts clean. This deletes data — it re-checks the archive first and aborts if anything is missing.

5Open new session

Create the 2027-28 academicSessions document, mark it active, and deactivate the old session so the app points at the new year.
