# Work Log — 2026-05-29

Session covering report-card improvements (LKG–X), student portal fixes, a full
data reset, and demo seeding. All changes deployed to
`https://st-francis-school-a3e7e.web.app`.

## 1. Kindergarten (LKG / SKG) report cards — categories + grades-only

**Problem:** LKG/SKG cards listed all 74 criteria individually → overflowed the
single fixed-height landscape page (only ~35 of 78 rows visible, two subjects
clipped entirely).

**Changes:**
- Added a `category` field to every criterion in the 4 KG criteria files
  (`assessment-app/data/criteria/{literacy,numeracy,rhymes-stories,arts}.json`):
  - Literacy / Numeracy → Writing Skills · Work Habits · Speaking Skills · Identification
  - Rhymes & Stories → Rhymes·Work Habits · Rhymes·Learning · Stories·Listening · Stories·Learning
  - Arts/Colouring → Work Habits · Learning
- `report-card-print.js`: LKG/SKG now render **category-grouped** rows (like
  Class I/II) and **grades only — no numeric scores** (`gradesOnly` path:
  4-column table, "Overall Grade"/"Year Grade" panels). Header reads
  "Kindergarten Section".
- Threaded `category` through the real pipeline so saved cards match:
  `report-card-aggregator.js` + `report-card-builder.js`.

**Result:** KG cards fit one page, grouped + grade-based.

## 2. Performance-rooted teacher remarks

Confirmed the report card builder uses `report-card-remark-engine.js`
(opener by overall grade, strongest/weakest subject, trend-based closer) — the
remark is derived from the student's actual results, not random text.

## 3. Attendance pulled live (NEW)

- New `assessment-app/services/report-card-attendance.js` — reads a term's
  `attendance_monthly/{classNum}_{yyyy}_{mm}` snapshots, sums **present + late**
  as present days and `working_days` as total. Late counts as present.
- Wired into `report-card-builder.js` (was hard-coded `null`). Falls back to
  blank if no snapshot exists.

## 4. Student "My Progress" (LKG–II) fixes

- **Permission:** `firestore.rules` now lets a student read
  `assessment_sessions` for **their own class** (mapped to the session label
  format). Previously staff-only → every student got "Missing or insufficient
  permissions".
- **Criteria-path bug:** `student-progress-engine.js` fetched
  `data/criteria/…` from the root portal page (404). Now prefixed with
  `assessment-app/`.
- **Refactor (non-breaking):** exported `computeProgressFromSessions()`
  (engine) and `renderStudentProgress()` (view) so the view can be driven with
  in-memory data (used by the demo).
- **Hide tab for III–X:** `index.html` + `student-progress-init.js` hide the
  "My Progress" sidebar item for classes 3–10 (assessment system is LKG–II only).
- **Mobile fix:** `report-card-styles.css` — individual criteria now show
  `name · bar · %` on phones (only the band chip is dropped); previously the %
  was hidden.

## 5. Class III–X report cards → visible to students after release (NEW)

Class III–X marks live in the `marks` collection (Sfs-report-card system), not
`report_cards`; the student-facing view only read `report_cards`, so III–X
students could never see their card.

- New `report-card-student-iiix.js` — for a III–X student: maps class
  (Arabic → Roman `classId`), reads released `marks/{classId}_FT` (+ `_HY`),
  and renders via the existing III–X design (`Sfs-report-card/reportcard.html`
  + `render.js`) using the same `sessionStorage` bridge as the admin. New tab on
  desktop, in-portal iframe overlay on the APK. Shows "not released yet" until
  released.
- `report-card-student-view.js` branches by class (III–X → new module; LKG–II →
  existing `report_cards` flow).
- `firestore.rules`: new `ftReleasedForStudent()` helper lets a student read
  **both** term docs once the **Final Term** doc is released (release only flags
  `_FT`).
