## 1. Critical Bug Fix ‚Äî App Completely Broken (`_origShowPage is not a function`)

**Root cause:** Two orphaned closing brackets (`});` and `}`) were left in `script.js` after the SW update banner was removed in the previous session. This caused a syntax error that prevented `script.js` from executing entirely, meaning `showPage()` was never defined. `app-logic.js` then captured `undefined` as `_origShowPage`, crashing every button click.

**Fix:**
- Removed the stale `});` + `}` from the end of `script.js`
- Reverted `app-logic.js?v=18` ‚Üí `app-logic.js?v=17` in `index.html` (the `?v=18` cache-bust was an earlier failed attempt and was not the real cause)

**Files:** `script.js`, `index.html`

## 2. Android Back Button Fix (`back-button.js`)

**Problem:** Pressing the hardware/gesture back button on Android exited the PWA instead of navigating within the app.

**Root cause (previous attempt):** The inline history patch used `DOMContentLoaded` timing which was fragile ‚Äî `app-logic.js` (module) also wraps `window.showPage` during the same phase, creating race conditions.

**Fix:** Created new focused file `back-button.js` (per project memory rule ‚Äî new feature = new file):
- Uses `window.load` event ‚Äî fires after ALL modules have executed, captures the final `window.showPage` with no timing ambiguity
- Uses `history.pushState` (not `replaceState`) for the initial home entry ‚Äî provides a real back-stop so Android can't swipe out on the first press
- On every `popstate` back to home, re-pushes a new home entry ‚Äî user can never exit the app by pressing back from the home screen
- Removed the old inline script block from `index.html`
- Added `back-button.js` to SW cache shell (`sfs-v19`)

**Files created:** `back-button.js`
**Files modified:** `index.html`, `sw.js`
